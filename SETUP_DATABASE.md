# KaamConnect Database Setup Guide

This guide will help you set up the Appwrite collections and populate them with sample data.

## Collections to Create

### 1. Workers Collection
**Collection ID:** `workers`

**Fields:**
- `$id` (String) - Auto-generated by Appwrite
- `name` (String, Required) - Worker's full name
- `email` (String, Required) - Email address
- `phone` (String, Required) - Phone number
- `category` (String, Required) - Work category (electrician, plumber, etc.)
- `city` (String, Required) - Worker's city
- `rate` (Integer, Required) - Daily rate in INR
- `experience` (Integer) - Years of experience
- `available` (Boolean, Default: true) - Availability status
- `rating` (Float, Default: 0) - Average rating (0-5)
- `reviewsCount` (Integer, Default: 0) - Number of reviews
- `photo` (String) - URL to profile photo
- `languages` (String) - Languages spoken (comma-separated)
- `skills` (String) - Skills (comma-separated)
- `about` (String) - Bio/about section
- `userId` (String) - Link to Appwrite Auth user

### 2. Jobs Collection
**Collection ID:** `jobs`

**Fields:**
- `$id` (String) - Auto-generated
- `title` (String, Required) - Job title/description
- `category` (String, Required) - Work category
- `city` (String, Required) - Job location
- `rate` (Integer, Required) - Budget/hourly rate
- `duration` (String, Required) - Job duration (e.g., "1 Day", "1 Week", "Ongoing")
- `status` (String, Required, Default: "open") - Job status (open, closed, completed)
- `employerId` (String, Required) - Employer user ID
- `description` (String) - Detailed job description
- `applicationsCount` (Integer, Default: 0) - Number of applications
- `createdAt` (String) - ISO timestamp
- `updatedAt` (String) - ISO timestamp

### 3. Applications Collection
**Collection ID:** `applications`

**Fields:**
- `$id` (String) - Auto-generated
- `jobId` (String, Required) - Reference to job
- `workerId` (String, Required) - Reference to worker
- `status` (String, Default: "pending") - Application status (pending, accepted, rejected)
- `appliedAt` (String) - ISO timestamp
- `respondedAt` (String) - When employer responded

### 4. Reviews Collection
**Collection ID:** `reviews`

**Fields:**
- `$id` (String) - Auto-generated
- `workerId` (String, Required) - Worker being reviewed
- `employerId` (String, Required) - Employer who hired
- `rating` (Float, Required) - Rating (1-5)
- `comment` (String) - Review text
- `jobId` (String) - Reference to completed job
- `createdAt` (String) - ISO timestamp

## Steps to Add Data

### Option 1: Using Appwrite Console (GUI)

1. Go to [Appwrite Console](https://cloud.appwrite.io)
2. Select your project (KaamConnect)
3. Go to **Databases** → **69e9749100232561665e**
4. Create each collection with the schema above
5. Click **Create Document** and paste the sample data

### Option 2: Using Appwrite REST API

You can create collections and documents using curl commands:

```bash
# Create workers collection
curl -X POST https://sgp.cloud.appwrite.io/v1/databases/69e9749100232561665e/collections \
  -H "X-Appwrite-Project: 69d7693c0026a410e676" \
  -H "Content-Type: application/json" \
  -d '{
    "collectionId": "workers",
    "name": "Workers",
    "permissions": ["read(\"any\")", "write(\"user\")"]
  }'

# Add worker document
curl -X POST https://sgp.cloud.appwrite.io/v1/databases/69e9749100232561665e/collections/workers/documents \
  -H "X-Appwrite-Project: 69d7693c0026a410e676" \
  -H "Content-Type: application/json" \
  -d '{
    "documentId": "auto",
    "data": {
      "name": "Ramesh Kumar",
      "email": "ramesh@example.com",
      "phone": "9876543210",
      "category": "electrician",
      ...
    }
  }'
```

### Option 3: Using JavaScript (In Browser Console)

```javascript
// Add this to your app after authentication

import { databases, ID, appwriteConfig } from "./appwrite.js";

async function setupDatabase() {
  const workers = [
    {
      name: "Ramesh Kumar",
      email: "ramesh@example.com",
      phone: "9876543210",
      category: "electrician",
      city: "Lajpat Nagar",
      rate: 600,
      experience: 8,
      available: true,
      rating: 4.9,
      reviewsCount: 47
    },
    // ... more workers
  ];

  for (const worker of workers) {
    await databases.createDocument(
      appwriteConfig.databaseId,
      appwriteConfig.workersCollectionId,
      ID.unique(),
      worker
    );
  }

  console.log("Database setup complete!");
}

// Call: setupDatabase()
```

## Notes

- Replace `EMPLOYER_ID_*` and `WORKER_ID_*` with actual document IDs
- Use ISO format for all date fields: `new Date().toISOString()`
- For photos, use image URLs from Unsplash or upload to Storage bucket
- Make sure user has logged in as an employer before posting jobs
- Set correct permissions on collections for users to read/write

## Troubleshooting

**Collections not showing data:**
- Check if collection IDs match in `appwrite.js`
- Verify user is logged in
- Check browser console for errors with `[DB:...]` prefix

**Application errors:**
- Check that `appwrite.js` has correct endpoint, projectId, databaseId
- Verify collections are created with correct IDs
- Check Appwrite console for permission errors

**Empty database:**
- Collections may be created but empty
- Add sample data following options above
- Once you have users signing up, their data will be saved automatically
